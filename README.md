Связка из двух частей:

- **CLI `tinvest`** — полноценный инвестиционный клиент поверх
  [T-Invest API](https://developer.tbank.ru/invest/api):
  - портфель, реальная доходность (XIRR), структура (секторы/валюты/страны),
    календарь пассивного дохода, свободные деньги, операции с комиссиями;
  - котировки, история цены с бенчмарком (IMOEX), стакан, техиндикаторы,
    карточки любых активов, облигации с расчётом YTM и дюрации, дивиденды,
    фундаментал, прогнозы аналитиков;
  - скринеры облигаций (по YTM/сроку/риску) и акций (по фундаменталу)
    по всему справочнику с локальным кэшем;
  - новости, сделки инсайдеров, календарь отчётностей, сигналы стратегий,
    вотчлист;
  - торговля: заявки и стоп-заявки в песочнице и (с обязательным `--confirm`)
    на боевом счёте; в readonly мутации заблокированы кодом.
- **Скилл Claude Code `t-invest`** — превращает Claude в инвестиционного
  консультанта: на вопрос «как мой портфель?» Claude сам вызывает CLI
  (с флагом `--json`) и анализирует данные.

## Режимы и токены

CLI работает в трёх режимах, у каждого — собственный токен:

| Режим | Переменная | Что это |
|---|---|---|
| `sandbox` | `T_INVEST_TOKEN_SANDBOX` | песочница: виртуальный счёт, отдельный контур API |
| `readonly` | `T_INVEST_TOKEN_READONLY` | боевой счёт, токен «только чтение» (сделки невозможны) |
| `full` | `T_INVEST_TOKEN_FULL` | боевой счёт, реальные сделки; токен уровня «Торговля» (НЕ «Торговля и переводы» — переводы/выводы CLI не использует) |

Токены выпускаются в настройках Т-Инвестиций → «Токены T-Invest API» и
записываются в **единственный канонический файл** `~/.config/tinvest/.env`
(см. `.env.example` как шаблон):

```bash
mkdir -p ~/.config/tinvest
cp .env.example ~/.config/tinvest/.env   # и впишите нужные токены
chmod 600 ~/.config/tinvest/.env
```

CLI читает `.env` **только** из `~/.config/tinvest/.env` (не из cwd) — это
детерминированно и не зависит от того, из какой папки запущен скилл. Реальные
переменные окружения имеют приоритет (штатный способ для CI/контейнеров). Без
единого токена CLI сразу завершается с ошибкой `APP_TINVEST_TOKEN_MISSING`
(никаких значений по умолчанию).

Хосты API: используются действующие `invest-public-api.tinkoff.ru` /
`sandbox-invest-public-api.tinkoff.ru` — канонические адреса `tbank.ru`
из документации недоступны из текущего окружения (TLS-перехват на уровне
сети, проверено 2026-07-02).

### Активный режим и граница реальных денег

Две независимые вещи:

**Активный режим — персистентная памятка.** Скилл при активации фиксирует режим
(по умолчанию `readonly`); пока режим не выбран, команды с данными не выполняются
(`APP_TINVEST_SESSION_REQUIRED`). `readonly`, `sandbox` и `full` переключаются
свободно той же командой `session start --mode <режим>` — новую сессию открывать
не нужно. Источник правды — `session status` (агент сверяется с ним, а не с
памятью, в т.ч. после потери контекста). Состояние привязано к идентичности
сессии: по `TINVEST_SESSION_ID` (задаёт обёртка — работает под любым агентом и
per-command шеллами) либо к одному глобальному файлу на пользователя.

**Реальные деньги — гейт в окружении, а не в сессии.** Само наличие full-токена
НЕ разрешает торговать. Лестница гейтов мутаций (в коде, до обращения к API):

- `readonly` — любая сделка запрещена (`APP_TINVEST_TRADING_FORBIDDEN`);
- `sandbox` — виртуальные сделки свободны;
- `full` без `T_INVEST_ALLOW_TRADING` — только чтение, сделка →
  `APP_TINVEST_TRADING_DISABLED`;
- `full` с `T_INVEST_ALLOW_TRADING=true` — сделка требует `--confirm` (подпись
  человека на каждую заявку), иначе `APP_TINVEST_CONFIRM_REQUIRED`;
- `full` с `T_INVEST_STONKS_MODE=true` — сделки без подтверждений (автономно,
  осознанный опасный opt-in деплоя; предупреждение показывается один раз).

Честная оговорка: разделение режимов и гейт торговли — защита штатных путей
(флаг деплоя + подтверждение сделки), а не криптография; субъект с полным
доступом к окружению может включить флаг сам, но это осознанное явное действие.

Доходность к погашению облигаций (эффективная, XIRR по денежным потокам),
доходность к оферте и дюрация рассчитываются самим CLI — T-Invest API готовых
значений не отдаёт. Для флоатеров, амортизации и бессрочных выпусков расчёт
честно возвращает `null` с русским предупреждением о причине. Той же
математикой считается XIRR-доходность счёта в `performance` (с выбором
экономически осмысленного корня при чередовании вводов/выводов).

### Кэш справочников

Списки инструментов (сутки) и графики купонов (неделя) кэшируются в
`~/.config/tinvest/cache` — скринеры и `allocation` после первого прогона
работают за доли секунды. Кэш можно безопасно удалить целиком.

### Безопасность торговли

- `readonly` — мутации отклоняются кодом (`APP_TINVEST_TRADING_FORBIDDEN`)
  ещё до обращения к API; чтение заявок доступно.
- `full` — каждая мутация требует явного флага `--confirm`
  (`APP_TINVEST_CONFIRM_REQUIRED` без него); скилл ставит флаг только после
  подтверждения пользователем конкретной заявки.
- Ключ идемпотентности (`--order-id`) защищает от дублирования заявки при
  повторе после сетевого сбоя.

Диагностика: любая ошибка выводится по-русски со стабильным кодом
(`APP_TINVEST_...`); технические детали — при запуске с `TINVEST_DEBUG=1`.

## Подключение и работа

Скилл работает через ИИ-агента (Claude Code или совместимый): тело скилла
(`SKILL.md`) — набор инструкций, которым агент следует, а вся работа идёт через
самодостаточный CLI `scripts/tinvest.cjs`. Ниже — что нужно для запуска.

### 1. Требования
- **Node.js ≥ 20** — CLI исполняется через него. Это единственная внешняя
  зависимость: npm-пакеты и сборка не нужны, всё вшито в один файл.
- Агент, умеющий читать `SKILL.md` и запускать shell-команды.

### 2. Установка
Скачайте `t-invest.skill` — кнопкой в шапке или из
[Releases](https://github.com/nyxandro/t-invest-skill/releases). Это ZIP-архив:
его нужно **распаковать** в каталог скиллов, а не класть файлом (сам файл
Claude Code не загрузит):
```bash
unzip t-invest.skill -d ~/.claude/skills/
# итог: ~/.claude/skills/t-invest/{SKILL.md, scripts/tinvest.cjs, references/}
```
Claude Code подхватывает скилл из `~/.claude/skills/<имя>/` автоматически, без
правок в settings. Для раздачи многим можно переупаковать в плагин с
`.claude-plugin/plugin.json` и ставить через `/plugin install`.

### 3. Токены — куда прописать
CLI читает `.env` **только** из `~/.config/tinvest/.env` (не из текущей папки —
детерминированно, из любой директории запуска):
```bash
mkdir -p ~/.config/tinvest
cat > ~/.config/tinvest/.env <<'EOF'
T_INVEST_TOKEN_SANDBOX=      # токен песочницы
T_INVEST_TOKEN_READONLY=     # боевой счёт, уровень «Только просмотр»
T_INVEST_TOKEN_FULL=         # боевой счёт, уровень «Торговля» (НЕ «Торговля и переводы»)
EOF
chmod 600 ~/.config/tinvest/.env
```
Токены выпускаются в Т-Инвестициях → «Токены T-Invest API». Уровень — по режиму;
**«Торговля и переводы» не нужен**: CLI не делает переводов/выводов, только
сделки и чтение. Токен — секрет, в чат не отправляется. Реальные переменные
окружения имеют приоритет над файлом (для CI/контейнеров).

### 4. Реальные сделки выключены по умолчанию
Даже с full-токеном сделки заблокированы, пока в env не задан флаг:
```bash
T_INVEST_ALLOW_TRADING=true   # разрешить реальные сделки (каждая — с --confirm)
T_INVEST_STONKS_MODE=true     # ОПАСНО: сделки БЕЗ подтверждений (автономно)
```
Лестница гейтов (в коде, до обращения к API):
- `readonly` — любые сделки запрещены;
- `sandbox` — виртуальные деньги, торговля свободна;
- `full` без `ALLOW_TRADING` — только чтение (сделка → `APP_TINVEST_TRADING_DISABLED`);
- `full` + `ALLOW_TRADING` — сделка требует `--confirm` (подпись человека, иначе `APP_TINVEST_CONFIRM_REQUIRED`);
- `full` + `STONKS_MODE` — сделки без подтверждений (осознанный опасный opt-in).

**Гарантия безопасности денег:** уровень токена «Торговля» и отсутствие в коде
методов перевода/вывода означают, что деньги **нельзя вывести со счёта** —
возможны только операции с бумагами внутри самого счёта.

### 5. Рабочий цикл
1. Проверить среду: `node --version` (≥ 20).
2. Узнать состояние:
   `node ~/.claude/skills/t-invest/scripts/tinvest.cjs session status --json`
   (активный режим, наличие токенов, гейт торговли, stonks).
3. Если режим не выбран — спросить пользователя и зафиксировать:
   `... --mode <readonly|sandbox|full> session start` (по умолчанию `readonly`).
   Без выбранного режима команды с данными отклоняются (`APP_TINVEST_SESSION_REQUIRED`).
4. Работать: `portfolio`, `quote`, `bond`, `screen bonds`, `order preview/buy/…`
   (полный список — в разделе «Использование» выше). Всегда с флагом `--json`.
5. Переключение режима — свободно, той же командой `session start --mode X`.

Правила поведения (дисциплина режимов, подтверждение каждой сделки, интерпретация
полей JSON) описаны в `SKILL.md` и `references/` — агент обязан им следовать.

### 6. Аудит и несколько агентов
- Каждая мутация (реальная и в песочнице) дописывается в
  `~/.config/tinvest/trades.log` (время, режим, бумага, лоты, цена, номер заявки,
  ключ идемпотентности) — виден весь след действий агента.
- Для параллельных агентов/сессий задай `TINVEST_SESSION_ID` — активный режим
  привяжется к нему (иначе используется один общий режим на пользователя).
- Разрешения на запуск CLI настраиваются на стороне агента (в Claude Code —
  `~/.claude/settings.json` → `permissions.allow`); скилл их не навязывает.
