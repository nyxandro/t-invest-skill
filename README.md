# t-invest-skill — CLI + скилл-консультант для Т-Инвестиций

Связка из двух частей:

- **CLI `tinvest`** — полноценный инвестиционный клиент поверх
  [T-Invest API](https://developer.tbank.ru/invest/api):
  - портфель, реальная доходность (XIRR), структура (секторы/валюты/страны),
    календарь пассивного дохода, свободные деньги, операции с комиссиями;
  - котировки, история цены с бенчмарком (IMOEX), стакан, техиндикаторы,
    карточки любых активов, облигации с расчётом YTM и дюрации, дивиденды,
    фундаментал, прогнозы аналитиков;
  - скринеры облигаций (по YTM/сроку/риску) и акций (по фундаменталу)
    по всему справочнику с локальным кэшем;
  - новости, сделки инсайдеров, календарь отчётностей, сигналы стратегий,
    вотчлист;
  - торговля: заявки и стоп-заявки в песочнице и (с обязательным `--confirm`)
    на боевом счёте; в readonly мутации заблокированы кодом.
- **Скилл Claude Code `t-invest`** — превращает Claude в инвестиционного
  консультанта: на вопрос «как мой портфель?» Claude сам вызывает CLI
  (с флагом `--json`) и анализирует данные.


## Режимы и токены

CLI работает в трёх режимах, у каждого — собственный токен:

| Режим | Переменная | Что это |
|---|---|---|
| `sandbox` | `T_INVEST_TOKEN_SANDBOX` | песочница: виртуальный счёт, отдельный контур API |
| `readonly` | `T_INVEST_TOKEN_READONLY` | боевой счёт, токен «только чтение» (сделки невозможны) |
| `full` | `T_INVEST_TOKEN_FULL` | боевой счёт, полный доступ (задел под торговые команды v2) |

Режим задаётся флагом `-m/--mode`; если заполнен ровно один токен — режим
определяется автоматически. При нескольких заполненных токенах без `--mode`
CLI требует явного выбора (`APP_TINVEST_MODE_AMBIGUOUS`) — счёт не угадывается.

Токены выпускаются в настройках Т-Инвестиций → «Токены T-Invest API» и
записываются в `.env` (см. `.env.example`):

```bash
# либо в папке проекта
cp .env.example .env   # и впишите нужные токены

# либо глобально — чтобы CLI работал из любой папки
mkdir -p ~/.config/tinvest && $EDITOR ~/.config/tinvest/.env
```

Порядок поиска: переменные окружения → `.env` текущей папки →
`~/.config/tinvest/.env`. Без единого токена CLI сразу завершается с ошибкой
`APP_TINVEST_TOKEN_MISSING` (никаких значений по умолчанию).

Хосты API: используются действующие `invest-public-api.tinkoff.ru` /
`sandbox-invest-public-api.tinkoff.ru` — канонические адреса `tbank.ru`
из документации недоступны из текущего окружения (TLS-перехват на уровне
сети, проверено 2026-07-02).

### Фиксация режима на сессию (граница безопасности)

Скилл-консультант при первой активации спрашивает у пользователя режим и
фиксирует его на сессию — дальше CLI **на уровне кода** отклоняет команды
других режимов (`APP_TINVEST_MODE_LOCKED`), даже если агента уговорили
«переключиться».

Замок привязан к **процессу запущенной сессии Claude Code** (поиск предка
`claude` по `/proc` с защитой от переиспользования PID): закрыл сессию —
процесс умер — замок недействителен, новая сессия заново выбирает режим.
Никаких TTL и ручных команд для смены режима не нужно. Замки лежат в
`~/.config/tinvest/sessions/` (файл на сессию, параллельные сессии не
конфликтуют, мёртвые замки зачищаются автоматически).

```bash
node dist/cli.js --mode sandbox session start        # зафиксировать режим текущей сессии
node dist/cli.js --mode full session start --acknowledge-trading  # full требует явного подтверждения
node dist/cli.js session status                      # активная сессия
node dist/cli.js session end                         # ручной сброс (обычно не нужен)
```

Честная оговорка: это защита штатных путей (defense in depth), а не криптография —
субъект с полным доступом к файловой системе может удалить замок, но это уже
явное, заметное действие, а не «уговорил агента».

### Песочница

Счёт в песочнице создаётся только через API. Первичная инициализация:

```bash
node dist/cli.js --mode sandbox sandbox init                    # счёт + 1 000 000 ₽
node dist/cli.js --mode sandbox sandbox init --amount 500000    # своя сумма
```

## Использование

```bash
# Портфель и аналитика
node dist/cli.js accounts               # список счетов
node dist/cli.js portfolio              # портфель: позиции, стоимость, P/L
node dist/cli.js performance            # реальная доходность: XIRR, вложено/выведено, комиссии
node dist/cli.js allocation             # структура: классы/секторы/валюты/страны, концентрация
node dist/cli.js income                 # календарь купонов и дивидендов на год
node dist/cli.js cash                   # свободные деньги и блокировки
node dist/cli.js operations -d 90       # операции за 90 дней (с комиссиями)

# Инструменты и рынок
node dist/cli.js quote SBER             # последняя цена по тикеру
node dist/cli.js search "золото"        # поиск инструментов
node dist/cli.js instrument NVTK        # карточка любого актива + статус торгов
node dist/cli.js history SBER --vs IMOEX  # динамика за год против индекса
node dist/cli.js orderbook SBER         # стакан: спред и ликвидность
node dist/cli.js tech GAZP              # RSI, SMA(20/50), MACD от API
node dist/cli.js bond RU000A10BPZ1      # карточка облигации: купоны, оферта, YTM, дюрация
node dist/cli.js dividends SBER         # дивиденды: объявленные, история, TTM-доходность
node dist/cli.js fundamentals SBER      # P/E, ROE, долг/EBITDA, дивидендные метрики, рост
node dist/cli.js forecast SBER          # консенсус аналитиков и целевые цены

# Скринеры (первый прогон греет кэш, дальше — доли секунды)
node dist/cli.js screen bonds --ytm-min 18 --years-min 1 --years-max 3 --risk-max moderate
node dist/cli.js screen shares --pe-max 5 --roe-min 15 --sort pe

# Информация и идеи
node dist/cli.js news GAZP              # новости по бумаге (или вся лента без тикера)
node dist/cli.js insiders GAZP          # сделки инсайдеров
node dist/cli.js reports SBER           # календарь отчётностей
node dist/cli.js signals --ticker GAZP  # сигналы аналитических стратегий
node dist/cli.js favorites              # вотчлист из приложения с ценами

# Торговля (sandbox — свободно; full — каждая мутация требует --confirm)
node dist/cli.js order preview SBER -q 2            # оценка: сумма, комиссия, доступные лоты
node dist/cli.js order buy SBER -q 2 --price 300    # лимитная заявка (без --price — рыночная)
node dist/cli.js order list / status <id> / cancel <id> / replace <id> -q 2 --price 305
node dist/cli.js stop-order set SBER -q 2 --type take-profit --stop-price 350
node dist/cli.js stop-order list / cancel <id>

node dist/cli.js portfolio --json       # машиночитаемый вывод (использует Claude)
```

Доходность к погашению облигаций (эффективная, XIRR по денежным потокам),
доходность к оферте и дюрация рассчитываются самим CLI — T-Invest API готовых
значений не отдаёт. Для флоатеров, амортизации и бессрочных выпусков расчёт
честно возвращает `null` с русским предупреждением о причине. Той же
математикой считается XIRR-доходность счёта в `performance` (с выбором
экономически осмысленного корня при чередовании вводов/выводов).

### Кэш справочников

Списки инструментов (сутки) и графики купонов (неделя) кэшируются в
`~/.config/tinvest/cache` — скринеры и `allocation` после первого прогона
работают за доли секунды. Кэш можно безопасно удалить целиком.

### Безопасность торговли

- `readonly` — мутации отклоняются кодом (`APP_TINVEST_TRADING_FORBIDDEN`)
  ещё до обращения к API; чтение заявок доступно.
- `full` — каждая мутация требует явного флага `--confirm`
  (`APP_TINVEST_CONFIRM_REQUIRED` без него); скилл ставит флаг только после
  подтверждения пользователем конкретной заявки.
- Ключ идемпотентности (`--order-id`) защищает от дублирования заявки при
  повторе после сетевого сбоя.

Диагностика: любая ошибка выводится по-русски со стабильным кодом
(`APP_TINVEST_...`); технические детали — при запуске с `TINVEST_DEBUG=1`.

## Разработка

```bash
npm test          # vitest, тесты рядом с кодом (*.test.ts)
npm run lint      # eslint
npm run typecheck # tsc --noEmit
npm run build     # сборка в dist/
```

Архитектура: `src/api` — REST-клиент T-Invest API, типы и деньги (units/nano);
`src/commands` — логика команд (клиент инъецируется, тестируется на моках из
`src/api/mocks`); `src/format` — табличный вывод; `src/config` — константы и
fail-fast разрешение секрета; `src/cli.ts` — commander-обвязка и граница ошибок.

## Дисклеймер

Инструмент предоставляет данные и аналитику. Ответы консультанта не являются
индивидуальной инвестиционной рекомендацией.
